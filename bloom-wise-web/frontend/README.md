# Подоконник — веб-приложение для заказа комнатных растений

Этот репозиторий содержит исходный код витрины магазина комнатных растений «Подоконник». Ниже описано, что нужно установить, как запустить проект и как менять содержимое, не погружаясь глубоко в программирование.

---

## 1. Что необходимо на компьютере

- **Node.js** версии 18 или выше. Скачать можно с официального сайта: <https://nodejs.org/>.
- **npm** устанавливается вместе с Node.js и нужен для запуска команд.
- Рекомендуется поставить текстовый редактор (например, Visual Studio Code: <https://code.visualstudio.com/>), но можно обойтись и блокнотом.
- Для загрузки обновлений из интернета пригодится **Git** (<https://git-scm.com/>), но если изменения будут делаться только локально — необязательно.

Проверить, что всё установлено, можно в командной строке (PowerShell или Терминал):

```sh
node -v
npm -v
```

Если обе команды показывают номер версии, всё готово.

## 3. Как запустить проект локально

1. Скачайте или распакуйте архив с проектом в удобную папку.
2. Откройте эту папку в терминале.
3. Установите зависимости (это одна команда, всё поставится автоматически):

   ```sh
   npm install
   ```

4. Запустите локальный сервер разработки:

   ```sh
   npm run dev
   ```

5. Терминал покажет адрес вида `http://localhost:5173`. Откройте его в браузере — увидите рабочую версию сайта.

### Полезные команды

- `npm run dev` — запустить сайт в режиме разработки (перезапускается сам при каждом изменении файлов).
- `npm run build` — собрать оптимизированную версию для публикации. Результат появится в папке `dist`.
- `npm run preview` — посмотреть собранную версию перед публикацией.
- `npm run lint` — проверить код на ошибки оформления (может быть полезно, если изменения делает разработчик).

## 4. Описание папок и файлов

- `index.html` — главная HTML-страница. Обычно менять не нужно, за исключением мета-тегов и SEO-информации.
- `public/` — статические файлы, которые отдаются как есть (например, `robots.txt`, `favicon.ico`). Сюда можно добавлять изображения, которые не требуют обработки.
- `src/` — основная рабочая папка:
  - `main.tsx` — входная точка приложения. Здесь подключается общий макет, маршруты и глобальные стили.
  - `App.tsx` — корневой компонент, в котором заданы общие элементы интерфейса и навигация.
  - `App.css`, `index.css` — глобальные стили. `index.css` подключает Tailwind, а `App.css` содержит дополнительные настройки.
  - `assets/` — изображения, иконки и PWA-манифест. Если нужно заменить логотип или картинку — положите сюда новый файл и обновите путь в коде.
  - `components/` — повторно используемые элементы. Здесь есть:
    - общие компоненты (`Footer.tsx`, `NavigationMenu.tsx`, `LeafDecoration.tsx`, `PWAInstallPrompt.tsx`);
    - папка `ui/` со стандартными компонентами shadcn/ui (кнопки, карточки, формы и т.п.). Их можно использовать повторно во всём приложении.
  - `hooks/` — вспомогательные функции для React (например, `use-mobile.tsx` определяет мобильное устройство, `use-toast.ts` показывает уведомления).
  - `lib/utils.ts` — вспомогательные утилиты (например, объединение CSS-классов).
  - `pages/` — отдельные страницы сайта (`Index.tsx`, `Catalog.tsx`, `Cart.tsx`, `Profile.tsx` и т.д.). Чтобы изменить содержимое конкретной страницы, откройте соответствующий файл.
  - `vite-env.d.ts` — служебный файл TypeScript, трогать не нужно.
- `components.json` — конфигурация shadcn/ui, необходима, если добавлять новые компоненты через их CLI.
- `tailwind.config.ts` — настройки Tailwind (цвета, шрифты, пути до файлов). Здесь можно менять тему и добавлять собственные классы.
- `postcss.config.js` и `eslint.config.js` — настройки инструментов сборки и проверки кода.
- `vite.config.ts` — общая конфигурация Vite и PWA (иконки, кэширование, поведение приложения).

## 5. Как изменить стили

1. **Простые изменения** (цвета текста, фон, отступы) можно делать прямо в файлах компонентов, изменяя классы Tailwind. Например: `className="bg-green-500 text-white"`.
2. Если нужно часто использовать одинаковые стили, добавьте их в `tailwind.config.ts` в раздел `theme.extend`, а затем применяйте как обычные классы.
3. Для глобальных правок (например, смена базового шрифта) отредактируйте `src/index.css`.
4. Дополнительные инструкции по Tailwind: <https://tailwindcss.com/docs/utility-first>.
5. Документация shadcn/ui с примерами компонентов: <https://ui.shadcn.com/docs/components>.

После изменения файлов Tailwind автоматически пересоберёт стили, если запущен `npm run dev`.

## 6. Как добавить новый элемент на страницу

1. Откройте нужную страницу в папке `src/pages`. Например, чтобы добавить блок на главную, редактируйте `src/pages/Index.tsx`.
2. Найдите место, куда нужно вставить новый элемент, и запишите HTML-разметку в стиле JSX. Можно копировать существующие блоки как шаблон.
3. Чтобы использовать готовые кнопки, карточки, формы, подключите компоненты из `src/components/ui`. Например:

   ```tsx
   import { Button } from "@/components/ui/button";

   <Button variant="default">Нажми меня</Button>
   ```

4. Если элемента нет, а вы хотите его использовать повторно в нескольких местах:
   - создайте файл в `src/components`,
   - опишите там JSX-компонент,
   - подключите его на нужных страницах.
5. Когда добавляете изображения, положите их в `src/assets` и используйте `import` или прямой путь `/src/assets/...`.

Всё приложение устроено модульно, поэтому изменения в одной странице не ломают остальные.

## 7. Сборка и публикация

1. Запустите `npm run build`. Появится папка `dist/` с готовыми файлами.
2. Содержимое `dist/` можно загрузить на любой статический хостинг (например, Netlify, Vercel, GitHub Pages или на собственный сервер).
3. Если используется PWA-функциональность, не забудьте проверить работу офлайн-режима.

Рекомендации по деплою Vite: <https://vite.dev/guide/static-deploy.html>.
